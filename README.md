# 五子棋 (Gomoku) — 双人对战 & 远程联机

基于 Python Tkinter 的桌面五子棋游戏，支持本机双人对战和 TCP/IP 远程联机对战，黑方严格实现 Renju 风格禁手规则。

---

## 功能特性

- **本地双人对战** — 在同一台电脑上轮流落子
- **远程联机对战** — 基于 TCP Socket，支持主机/客户端模式
- **黑方 Renju 禁手** — 三三禁手、四四禁手、长连禁手，检测算法规范严谨
- **成五优先原则** — 同时形成五连与禁手时，禁手失效，五连获胜
- **白方无禁手** — 按普通五子棋规则，连成 5 子或以上即获胜
- **落子音效** — 支持 MP3 或合成 WAV 音效
- **对局记录保存** — 落子步骤可导出为 JSON 文件
- **远程联机保活** — 心跳检测与超时断线自动处理
- **协议版本校验** — 确保联机双方版本一致

## 规则详细介绍

### 基本规则

- 15 × 15 标准棋盘
- 黑方先手，黑白轮流落子
- 任意一方先在 **横、竖、斜** 任一方向上连成 **恰好 5 子**，即获胜
- 棋盘完全下满且无人获胜，则为平局

### 黑方禁手规则

黑方受以下禁手限制。触发禁手的落子 **直接判负**，不可撤销。

#### 长连禁手 (Overline)

- 黑方落子后，在任一方向形成 **6 子或以上的连续同色棋子**，即为长连禁手
- 例如：棋盘上已有 5 颗黑子连成一线，黑方再下一子使之变为 6 连，触发长连禁手

#### 四四禁手 (Four-Four)

- 黑方一子落下后，**同时形成两个或以上** 的"四"（长度为 4 的黑子连通线段，且至少一端紧邻空位）
- **关键规则**：落子点 **必须在每一个四中**。棋盘上已经存在但与当前落子无关的"四"，不计入禁手判定
- 例如：落子同时在横向和纵向各形成一个"四"，即触发四四禁手

#### 三三禁手 (Three-Three)

- 黑方一子落下后，**同时形成两个或以上** 的"活三"
- **活三 (Live Three)** 的定义较为严格：
  1. 长度为 3 的黑子连通线段
  2. **两端紧邻位均为空**（不被对方棋子或棋盘边界阻挡）
  3. **至少有一端在再往外一格也为空**（保证该活三可以拓展为活四）
- **关键规则**：落子点 **必须在每一个活三中**
- 例如：落子同时在横向和纵向各形成一个满足上述条件的活三，即触发三三禁手

#### 禁手检测逻辑

1. 检测是否形成 **恰好 5 连** → 是则黑方获胜，**跳过所有禁手检测**
2. 检测是否 **长连**（≥6） → 是则禁手，黑方判负
3. 检测是否 **四四** → 是则禁手，黑方判负
4. 检测是否 **三三** → 是则禁手，黑方判负
5. 以上均未触发，落子合法

#### 示例说明

| 局面 | 结果 | 说明 |
|------|------|------|
| 黑子恰好 5 连 | ✅ 黑方获胜 | 五连优先，无视其他禁手 |
| 黑子 6 连或以上 | ❌ 长连禁手 | 即使同时形成五连，也优先判五连获胜 |
| 一子同时形成两个"四" | ❌ 四四禁手 | 两个"四"都必须包含该落子点 |
| 一子同时形成两个"活三" | ❌ 三三禁手 | 两个"活三"都必须包含该落子点 |
| 一个活三 + 一个眠三 | ✅ 合法 | 眠三不计入三三禁手 |

### 白方规则

- 白方 **没有任何禁手限制**
- 连成 5 子或以上即获胜（≥6 连同样算胜）

## 运行方式

### 基本运行

```bash
python gomoku.py
```

### 本地双人模式

1. 启动程序，点击 **「本地双人」** 按钮
2. 黑方先手，点击棋盘交叉点落子
3. 状态栏显示当前轮到哪一方
4. 触发禁手或连五时，程序会弹出结果提示

### 远程联机模式

1. 启动程序，点击 **「远程联机」** 按钮
2. 在弹出的窗口中，**主机方** 设置端口（默认 5000），点击 **「作为主机」**，等待对方连接
3. **客户端方** 输入主机 IP 地址和端口，点击 **「加入对局」**
4. 双方连接成功后自动开始对局
5. 每局结束后，下一局双方自动交换黑白方
6. 任一方点击 **「重新开始」** 可发起重置（需对方同步确认）

### 保存对局记录

- 对局过程中点击 **「保存对局」**，可将当前对局的落子记录导出为 JSON 文件
- 记录包含：步数、棋子颜色、落子坐标、是否合法、时间戳

## 技术细节

- **GUI 框架**：Python Tkinter，纯标准库实现
- **网络通信**：TCP Socket，自定义文本协议（HELLO / MOVE / RESULT / RESET / PING / PONG）
- **心跳保活**：每 5 秒发送 PING，15 秒无响应判定为断线
- **协议版本**：使用 `PROTOCOL_VERSION` 校验，版本不匹配时拒绝连接
- **防重复重置**：使用 `session_id` 机制，避免网络抖动导致重复重置
- **禁手算法**：遍历棋盘上所有同色连通线段，仅统计包含落子点的线段，确保严格符合 Renju 规则
